Calorify

C++ / JavaScript / Python

Calorify is a full-stack nutrition-tracking application that fuses real-time computer vision with custom data structures for sub-second food recognition and macro analysis. OpenCV handles frame-by-frame food scanning, while a ChatGPT integration generates personalized dietary recommendations based on user profiles.

Architecture: A Trie-based search index for O(k) food lookups, a HashMap for constant-time macronutrient retrieval, and a BST for ranked recommendation sorting — all hand-implemented without standard library containers.
Pipeline: Camera capture → OpenCV preprocessing → classification → nutritional data lookup → profile-aware calorie/macro calculation → recommendation engine.
Technical Highlights: Custom memory management for data structure nodes, efficient serialization for user profile persistence, and modular architecture enabling independent component testing.

Image Concatenation: UNIX Process Management

C

A systems-level C project implementing concurrent image processing through UNIX process orchestration. The parent process forks worker children, establishes bidirectional IPC via pipe(), and coordinates execution through exec() program replacement — mirroring real-world pipeline architectures.

Concurrency Model: Parent-child process hierarchy with non-blocking I/O, graceful signal propagation (SIGCHLD, SIGTERM), and proper zombie reaping to prevent resource leaks.
Systems Engineering: Raw file descriptor management, custom signal handlers with sigaction(), error-resilient process lifecycle control, and deterministic cleanup sequences across forked process trees.
Reliability: Every system call is guarded by error handling, with cascading cleanup on failure to ensure no orphaned processes or dangling file descriptors.

Hand Hero

C++ / Arduino / OpenCV

An embedded assistive-technology system that bridges computer vision with real-time hardware control. OpenCV processes webcam frames to detect and classify hand gestures, then transmits commands over a serial protocol to an Arduino microcontroller driving a DC motor, piezo buzzer, and indicator LEDs.

Real-Time Pipeline: Frame capture at 30fps → background subtraction → contour detection → convexity-defect analysis → gesture classification → serial command dispatch — all within a 33ms per-frame budget.
Embedded Control: Arduino firmware implements a state machine consuming serial commands, with PWM-based motor speed control, debounced input handling, and watchdog-style timeout for safety cutoff.
Integration: Modular sensor architecture allows hot-swapping peripherals, with a bidirectional serial feedback channel for real-time system monitoring and diagnostics.

AI Chess Engine with Neural Network

Python / TensorFlow

A chess engine combining deep learning evaluation with classical game-tree search. A TensorFlow neural network scores board positions across multiple features — material balance, king safety, pawn structure, and center control — while the search layer selects moves via Minimax, Negamax, and Alpha-Beta Pruning with configurable depth.

Search Optimization: Alpha-Beta pruning reduces the branching factor from ~35 to ~6 at depth 4, enabling real-time move selection. Move ordering heuristics (captures first, killer moves) maximize cutoff rates.
Neural Architecture: Fully connected layers with ReLU activations, trained on labeled board evaluations. The network replaces the traditional handcrafted evaluation function, yielding stronger positional play.
Modes: Human vs AI, AI vs AI (for architecture benchmarking), full move history with undo, and hot-swappable evaluation functions for A/B testing different network architectures.

Mo Money

C++

A production-grade C++ financial processing engine that ingests ETF transaction streams, computes Adjusted Cost Base (ACB) in real time, and generates capital gains/loss (CGL) reports compliant with Canada Revenue Agency requirements.

Data Architecture:

  • Transaction Class: Encapsulates ticker, date, type, shares, and value with RAII-managed linked-list pointers. Overloaded comparison operators enable custom sort by date and type for deterministic processing order.
  • History Class: Owns the transaction linked list lifecycle — insertion, chronological sorting, ACB/CGL propagation, and formatted output. Destructor guarantees zero memory leaks across arbitrary transaction volumes.
Memory Model: Manual heap management via raw pointers with disciplined ownership semantics — no smart pointers by design, enforcing explicit allocation/deallocation to build low-level memory fluency.
Processing Pipeline: File I/O parser → Transaction object construction → chronological sort → running ACB accumulation → per-sale CGL computation → formatted tax report output.
Outcome: Accurate, auditable ACB/CGL reporting for arbitrary ETF portfolios, stress-tested against edge cases (partial sales, same-day transactions, zero-cost-basis entries).
